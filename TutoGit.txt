	/* TUTORIEL POUR INITIALISER VOTRE PROJET GITHUB ET UTILISER LA MAGIE DES COMMANDES GIT */

-> Aller dans le repertoir github sur projet et cliquer sur fork en haut à droite. Cette manipulation va permettre de créer une copie du 
dossier dans votre propre github auquel vous aurez tout les droits de push et pull.
-> Aller ensuite dans votre repertoir fork, cliquez sur "code" et copier le lien https
-> Ensuite aller dans vos dossiers faites un clique droit puis "Git bash her" un invite de commande s'ouvre, entrez à présent les commandes suivantes : 

- git remote add upstream (collez ici le lien copié au préalable, le ctrl v ne fonctionne pas il faut faire clique droit puis coller)
ça devrais vous donner une ligne du style : "git remote add https://github.com/username/repositoryName.git"
- git fetch
- git pull upstream master
  ouvrez le fichier "Pull-Request-Verif.txt" Ajoutez votre nom puis enregistrez. 
- git add Pull-Request-Verif.txt
- git commit -m "add of my name for pull request verification"
- git push origin master 

Puis aller dans votre dossier sur github et faire un "new pull request" avec un titre de pull request adapté 
/!\ Les pulls request sont à faire uniquement lorsque votre fork compile /!\ 


/*-----------------------
      Commandes utiles
------------------------*/


- git pull upstream master (pour récupérer la dernière version du projet)
- git status (pour voir tout les fichiers modifiés)
- git add filename (commande obligatoire lorsque vous modifiez un fichier que vous voulez commit)
- git add ./ (commande pour ajouter tout les fichiers modifié)
- git commit -m "comments" (commande pour faire un commit à faire avant de push)
- git push (pour push vos modifications dans votre dossier fork github)
- git pull origin master (pour avoir la dernière version du projet de votre fork github)
- git stash (commande à faire en cas de conflit entre votre version actuelle et votre projet fork github)
